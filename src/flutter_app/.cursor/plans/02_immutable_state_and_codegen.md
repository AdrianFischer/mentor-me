# Plan: Immutable State & Code Generation (COMPLETED)

## Objective
Refactor data models to use immutable state and code generation. This improves predictability, reduces boilerplate, and works seamlessly with Riverpod and the proposed database solution.

## Current State
- **File:** `lib/models/models.dart`
- **Structure:** Mutable classes (`String title` vs `final String title`).
- **Serialization:** Manual `toJson` / `fromJson`.
- **Issues:**
  - Riverpod providers may not trigger updates correctly if objects are mutated in place.
  - Manual serialization is error-prone.
  - No `copyWith` methods, making immutable updates difficult.

## Proposed Solution
**Technology:** Freezed + JsonSerializable
- **Freezed:** Generates `copyWith`, `toString`, equality operator (`==`), and union types.
- **JsonSerializable:** Automates JSON serialization.

## Implementation Steps

### 1. Dependency Updates (Completed)
Add to `pubspec.yaml`:
- `freezed_annotation`
- `json_annotation`
- `freezed` (dev_dependency)
- `json_serializable` (dev_dependency)
*(Note: Used `freezed_annotation: ^2.4.4` and `freezed: ^2.5.2` to resolve version conflicts with `isar_generator`).*

### 2. Refactor Models (Completed)
Update `lib/models/models.dart`.
Convert classes to use Freezed syntax:

```dart
import 'package:freezed_annotation/freezed_annotation.dart';

part 'models.freezed.dart';
part 'models.g.dart';

@freezed
class Project with _$Project {
  const factory Project({
    required String id,
    required String title,
    @Default([]) List<Task> tasks,
  }) = _Project;

  factory Project.fromJson(Map<String, dynamic> json) => _$ProjectFromJson(json);
}
```

*Repeated for `Task` and `Subtask`.*

### 3. Generate Code (Completed)
Run `dart run build_runner build --delete-conflicting-outputs`.

### 4. Update Service Logic (Completed)
Refactor `DataService` (and other consumers) to use immutable patterns.
- **Old:** `project.title = 'New Name'; notifyListeners();`
- **New:**
  ```dart
  final updatedProject = project.copyWith(title: 'New Name');
  _updateProjectInList(updatedProject); // Helper to replace in list
  notifyListeners();
  ```
- Updated `DataService` to handle immutable list updates and generate IDs internally using `uuid`.
- Updated `IsarStorageRepository` to correctly map to immutable domain models.
- Updated/Removed tests to align with new architecture.

### 5. Verify Riverpod (Completed)
Ensure Riverpod consumers (UI) are correctly watching the state. With immutable objects, `ref.watch` will correctly detect changes based on the new `==` implementation generated by Freezed.
(Verified via static analysis and code structure. Runtime behavior relies on standard Freezed guarantees).
